# Security Policy

urlx handles security-sensitive operations including TLS connections, credential management, cookie storage, and proxy authentication. We take security seriously.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use [GitHub Security Advisories](https://github.com/jonwiggins/urlx/security/advisories) to report vulnerabilities privately. Click "Report a vulnerability" and provide:

- A description of the vulnerability
- Steps to reproduce or a proof of concept
- The potential impact

We will acknowledge reports within 72 hours and aim to provide a fix or mitigation plan within 14 days.

## Non-Security Bugs

For bugs that are not security-sensitive, please open a regular [GitHub issue](https://github.com/jonwiggins/urlx/issues).

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| 0.1.x   | No        |
